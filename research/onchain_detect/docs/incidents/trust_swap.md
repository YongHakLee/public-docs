## 1. 프로젝트 배경

### 1.1 TrustSwap이란?

TrustSwap은 DeFi 플랫폼으로, 다양한 금융 서비스를 제공합니다:

- **토큰 락업 (Token Locks)**: 토큰 베스팅 및 시간 잠금
- **스테이킹**: SWAP 토큰 스테이킹 보상
- **LaunchPad**: 새 프로젝트 토큰 세일 플랫폼
- **NFT LaunchPad**: NFT 프로젝트 런칭 지원
- **SmartSwap**: 탈중앙화 거래 기능

### 1.2 SWAP 토큰

- **토큰명**: TrustSwap (SWAP)
- **유형**: ERC-20
- **용도**: 거버넌스, 스테이킹, 수수료 할인

## 3. Excluded 데이터 분석

### 3.1 Excluded 비율이 높은 이유

TrustSwap 데이터의 Excluded 비율(10.4%)이 다른 프로젝트보다 높습니다:

| 프로젝트 | Excluded 비율 |
|---------|--------------|
| TrustSwap | 10.4% |
| NomadBridge | 5.2% |
| AudiusProject | 1.0% |
| SushiSwap | 2.4% |
| ShataCapital | 4.3% |

### 3.2 Excluded 가능 원인

1. **실패한 트랜잭션**: 가스 부족, 슬리피지 초과 등
2. **관련 없는 트랜잭션**: 분석 범위 외 활동
3. **불완전한 추적**: 실행 추적 수집 실패
4. **컨트랙트 업그레이드 관련**: 마이그레이션 트랜잭션

## 4. 정상 사용 패턴

### 4.1 주요 함수 호출

TrustSwap에서 관찰되는 일반적인 함수들:

| 함수 | 설명 |
|------|------|
| `stake()` | SWAP 토큰 스테이킹 |
| `unstake()` | 스테이킹 해제 |
| `claim()` | 보상 청구 |
| `lock()` | 토큰 락업 생성 |
| `unlock()` | 락업 해제 |
| `transfer()` | 토큰 전송 |

### 4.2 콜 트리 패턴

**일반적인 스테이킹 트랜잭션:**
```
CALL (stake)
├── STATICCALL (잔액 확인)
├── CALL (토큰 전송)
│   └── logs (Transfer 이벤트)
├── CALL (스테이킹 기록 업데이트)
└── logs (Staked 이벤트)
```

**토큰 락업 트랜잭션:**
```
CALL (lock)
├── CALL (토큰 전송)
├── STATICCALL (락업 조건 확인)
├── CALL (락업 생성)
│   └── state (스토리지 쓰기)
└── logs (TokensLocked 이벤트)
```

## 6. 데이터셋 활용

### 6.1 학습 데이터로서의 가치

TrustSwap 데이터는 다음 용도로 활용됩니다:

1. **정상 패턴 학습**: 악성 트랜잭션이 없어 순수한 정상 패턴 학습
2. **DeFi 활동 모델링**: 스테이킹, 락업 등 DeFi 활동 패턴
3. **클래스 균형**: 전체 데이터셋에서 정상 데이터 보충

### 6.2 분석 시 고려사항

1. **악성 대비 어려움**: 이 프로젝트만으로는 benign/malicious 비교 불가
2. **Excluded 처리**: 10.4%의 excluded 데이터 활용 방안 검토 필요
3. **도메인 특화**: 스테이킹/락업 중심의 패턴
